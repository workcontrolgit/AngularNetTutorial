# AI-Generated Dashboard Insights in Angular Material

## How to Prompt an LLM with Live Metrics and Display Smart Summaries on the Dashboard

Your dashboard already shows the numbers. This article teaches it to *explain* them.

By the end, the TalentManagement dashboard will automatically call the AI on page load — injecting the metrics it just fetched into a system prompt — and display a plain-English executive summary in a new `mat-card` at the top of the page.

## 🎯 What We're Building

A new **AI Insights** card at the top of the existing dashboard:

* Loads **automatically** when the dashboard metrics finish loading
* Constructs a system prompt from the already-loaded `DashboardMetrics` object
* Sends one `POST /api/v1/ai/chat` request with the metrics as context
* Displays the LLM's plain-English summary with a loading spinner and `smart_toy` icon
* Hidden entirely when `environment.aiEnabled` is `false` — zero impact on readers without Ollama

**Why `chat` instead of `hr-insight`?**

The `hr-insight` endpoint (Article 6.2) fetches fresh data from the database on each call. The dashboard already has the metrics in memory — fetching them again from the DB would be redundant. Instead, this article demonstrates **client-side prompt injection**: serializing the in-memory `DashboardMetrics` object as JSON and injecting it into the `systemPrompt` parameter that `AiService.chat()` already accepts. The LLM generates its summary from the data the frontend already holds — no extra DB round-trip needed.

This is a different and complementary pattern to the `hr-insight` approach: the backend owns the data access, and the frontend owns the prompt construction.

### Step 1: Update `dashboard.ts`

**Full diff — additions highlighted with comments:**

```typescript
import { Component, OnInit, OnDestroy, inject } from '@angular/core';  // add OnDestroy
import { Router } from '@angular/router';
import { CommonModule } from '@angular/common';
import { MatCardModule } from '@angular/material/card';
import { MatIconModule } from '@angular/material/icon';
import { MatButtonModule } from '@angular/material/button';
import { MatProgressSpinnerModule } from '@angular/material/progress-spinner';
import { MatListModule } from '@angular/material/list';
import { MatSnackBar, MatSnackBarModule } from '@angular/material/snack-bar';
import { BaseChartDirective } from 'ng2-charts';
import { ChartConfiguration } from 'chart.js';
import { Subject } from 'rxjs';                      // NEW
import { takeUntil } from 'rxjs/operators';          // NEW
import { PageHeader } from '@shared';
import { HasRoleDirective } from '../../shared/directives/has-role.directive';
import { DashboardMetrics, DepartmentMetric, PositionMetric, SalaryRangeMetric } from '../../models';
import { DashboardService, AiService } from '../../services/api';  // add AiService
import { environment } from '../../../environments/environment';   // NEW

@Component({
  selector: 'app-dashboard',
  standalone: true,
  templateUrl: './dashboard.html',
  styleUrl: './dashboard.scss',
  imports: [
    CommonModule,
    MatCardModule,
    MatIconModule,
    MatButtonModule,
    MatProgressSpinnerModule,
    MatListModule,
    MatSnackBarModule,
    BaseChartDirective,
    PageHeader,
    HasRoleDirective,
  ],
})
export class Dashboard implements OnInit, OnDestroy {  // add OnDestroy
  private dashboardService = inject(DashboardService);
  private aiService = inject(AiService);              // NEW
  private router = inject(Router);
  private snackBar = inject(MatSnackBar);
  private destroy$ = new Subject<void>();             // NEW

  loading = true;
  metrics: DashboardMetrics | null = null;

  // AI Insights state (NEW)
  aiEnabled = environment.aiEnabled;
  aiInsight = '';
  aiInsightLoading = false;
  aiInsightError = '';

  // ... existing chart properties remain unchanged ...

  ngOnInit(): void {
    this.loadDashboardMetrics();
  }

  ngOnDestroy(): void {          // NEW
    this.destroy$.next();
    this.destroy$.complete();
  }

  loadDashboardMetrics(): void {
    this.loading = true;

    this.dashboardService.getDashboardMetrics().subscribe({
      next: metrics => {
        this.metrics = metrics;
        this.prepareCharts(metrics);
        this.loading = false;
        if (this.aiEnabled) {   // NEW — trigger insight after metrics load
          this.loadAiInsight(metrics);
        }
      },
      error: error => {
        console.error('Error loading dashboard metrics:', error);
        this.showMessage('Error loading dashboard data');
        this.loading = false;
      },
    });
  }

  // NEW — build a system prompt from in-memory metrics and call the chat endpoint
  private loadAiInsight(metrics: DashboardMetrics): void {
    this.aiInsightLoading = true;
    this.aiInsightError = '';
    this.aiInsight = '';

    const systemPrompt = `You are an HR analytics assistant. Analyze the following workforce metrics and provide a concise executive summary (3-4 sentences) highlighting key observations, any notable patterns, and one actionable recommendation. Be specific — reference the actual numbers.

Workforce Metrics:
${JSON.stringify(metrics, null, 2)}`;

    const question = 'Provide a brief executive summary of the current workforce.';

    this.aiService
      .chat(question, systemPrompt)
      .pipe(takeUntil(this.destroy$))
      .subscribe({
        next: response => {
          this.aiInsight = response.reply;
          this.aiInsightLoading = false;
        },
        error: err => {
          this.aiInsightError =
            err?.error?.detail ?? 'AI insights unavailable. Is the API running with AiEnabled: true?';
          this.aiInsightLoading = false;
        },
      });
  }

  // ... existing private prepareCharts methods unchanged ...
  // ... existing navigate/showMessage methods unchanged ...
}
```

**Key design decisions:**

* **`if (this.aiEnabled)` before calling `loadAiInsight`** — The guard is in the TypeScript, not just the template. If `aiEnabled` is `false`, no HTTP request is ever made. Original tutorial readers never see a network error in the browser console.

* **`loadAiInsight` is `private`** — It is an internal implementation detail of the dashboard load sequence, not a public method to be called from the template. The template only reads the resulting state (`aiInsight`, `aiInsightLoading`, `aiInsightError`).

* **`JSON.stringify(metrics, null, 2)`** — Pretty-printing the metrics into the system prompt makes the structure readable to the LLM. Token cost is low: a typical `DashboardMetrics` object is under 500 tokens even with 10 departments.

* **`takeUntil(this.destroy$)`** — Same teardown pattern from Article 6.3. If the user navigates away before Ollama responds, the subscription is cancelled cleanly. Without this, the callback would attempt to update `aiInsight` on a destroyed component.

* **Independent loading state** — `aiInsightLoading` is separate from `loading`. The dashboard charts appear as soon as metrics load. The insights card shows its own spinner while Ollama is inferring — users see data immediately and the AI summary fills in shortly after.

### Step 2: Update `dashboard.html`

Add the AI Insights card **between the `<page-header />` and the loading spinner**. Placing it first in the DOM means it appears above the metric cards when it loads, making it a natural "read first" element.

```html
<page-header />

<!-- AI Insights Card (hidden when aiEnabled is false) -->
<mat-card *ngIf="aiEnabled" class="ai-insights-card">
  <mat-card-header>
    <mat-icon mat-card-avatar>smart_toy</mat-icon>
    <mat-card-title>AI Workforce Insights</mat-card-title>
    <mat-card-subtitle>Generated from live dashboard metrics</mat-card-subtitle>
  </mat-card-header>
  <mat-card-content>

    <!-- Loading -->
    <div *ngIf="aiInsightLoading" class="ai-insight-loading">
      <mat-spinner diameter="24"></mat-spinner>
      <span>Analyzing workforce data&hellip;</span>
    </div>

    <!-- Insight text -->
    <p *ngIf="!aiInsightLoading && aiInsight" class="ai-insight-text">
      {{ aiInsight }}
    </p>

    <!-- Error -->
    <p *ngIf="!aiInsightLoading && aiInsightError" class="ai-insight-error">
      <mat-icon>warning</mat-icon>
      {{ aiInsightError }}
    </p>

  </mat-card-content>
</mat-card>

<!-- Loading Spinner -->
<div *ngIf="loading" class="loading-spinner">
  <mat-spinner></mat-spinner>
</div>

<!-- ... rest of dashboard unchanged ... -->
```

**Key design decisions:**

* **`*ngIf="aiEnabled"` on the card** — The entire card is removed from the DOM when `aiEnabled` is `false`. There is no empty space, no spinner, no placeholder — readers without Ollama see no trace of the AI feature.

* **`diameter="24"` on the insight spinner** — A smaller spinner (24px vs the default 100px) fits naturally inline with text. The dashboard's main `<mat-spinner>` for metrics loading is full-size; using the same size for the AI insight would imply the entire page is blocked, which is misleading.

* **`&hellip;` in "Analyzing workforce data…"** — HTML entity rather than `...` to avoid lint warnings about ellipsis characters.

## 💡 Prompt Engineering: Why This Works

The system prompt pattern used here is worth understanding — it applies to any scenario where you want to ground an LLM in live application data.

**The pattern:**

```
You are a [domain expert]. Analyze the following [data type] and provide [specific output format]. [Constraints]. [Tone].

[Data label]:
[Serialized data]
```

**Applied to the dashboard:**

```
You are an HR analytics assistant. Analyze the following workforce metrics 
and provide a concise executive summary (3-4 sentences) highlighting key 
observations, any notable patterns, and one actionable recommendation. 
Be specific — reference the actual numbers.

Workforce Metrics:
{
  "totalEmployees": 247,
  "totalDepartments": 8,
  "newHiresThisMonth": 5,
  "averageSalary": 68500,
  "employeesByDepartment": [...]
  ...
}
```

**Why the constraints matter:**

* **"3-4 sentences"** — Without a length constraint, LLMs tend toward verbose bulleted lists. A dashboard card needs prose, not a report.
* **"Be specific — reference the actual numbers"** — Without this, the model may produce generic HR advice that ignores the data entirely ("Consider reviewing salary bands periodically…"). The instruction anchors the response to the actual values in the prompt.
* **"one actionable recommendation"** — Limits the model to a single call-to-action, making the insight useful for a busy HR manager who glances at the dashboard.

**Example output for a 247-employee company:**

> The workforce has grown steadily, with 5 new hires this month against a base of 247 employees across 8 departments. Engineering leads headcount at 72 employees, while Facilities remains understaffed at 4. The average salary of $68,500 is consistent with mid-market ranges for the technology sector. Given the concentration in Engineering and the thin coverage in Facilities, consider evaluating whether the Facilities team is resourced appropriately for the company's current size.

**Enable AI features:**

```bash
# In appsettings.json (ApiResources/TalentManagement-API/TalentManagementAPI.WebApi):
"FeatureManagement": {
  "AiEnabled": true
}

# In environment.ts (Clients/.../src/environments/):
```

**Start the stack:**

```bash
# Terminal 1: Ollama
ollama serve

# Terminal 2: .NET API
cd ApiResources/TalentManagement-API/TalentManagementAPI.WebApi
dotnet run

# Terminal 3: Angular
cd Clients/TalentManagement-Angular-Material/talent-management
npm start
```

Open `http://localhost:4200` → log in → the dashboard loads.

**Expected sequence:**

1. Dashboard metrics load → charts appear immediately
2. AI insights card shows "Analyzing workforce data…" spinner
3. After 5–15 seconds (Ollama inference time), the spinner is replaced by the executive summary

Set `aiEnabled: false` in `environment.ts` → hot-reload → navigate to `/`. The AI card is gone entirely. The dashboard looks identical to pre-Series 6.

## 🌟 Why This Pattern Scales

The client-side prompt injection pattern demonstrated here applies far beyond the dashboard.

**Any Angular component that fetches data can generate AI insights:**

* **Employee list** — Pass the current filtered result set to the LLM: *"These 12 employees are overdue for performance review. Summarize the risk."*
* **Project status page** — Inject milestone data: *"Three milestones are behind schedule. Identify the critical path risk."*
* **Financial report** — Serialize quarterly figures: *"Highlight variances greater than 10% and suggest investigation priorities."*

**The implementation is always the same:**

1. Fetch your data (already happening in `ngOnInit`)
2. Serialize to JSON for the system prompt
3. Call `AiService.chat(question, systemPrompt)`
4. Display the reply — with loading and error states

**Transferable skills:**

* **Prompt template construction** — Building prompts programmatically from typed Angular models, not hardcoded strings
* **Post-load async chaining** — Triggering a second async operation after the first completes, with independent loading states
* **Feature flag guarding** — Using `environment.aiEnabled` at both the TypeScript (`if (this.aiEnabled)`) and template (`*ngIf="aiEnabled"`) levels for defense in depth
